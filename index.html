Real-Time System Application 6

A proof-of-concept real-time embedded system demonstrating predictable timing, task scheduling, and hard deadline enforcement on FreeRTOS and ESP32.

Spring 2026 · UCF ESP32 · FreeRTOS Hard Real-Time · Deterministic
// 01 — Project Overview

What This Project Does

This is a proof-of-concept real-time system designed to prove that an embedded application can meet strict timing deadlines while executing concurrent tasks on a microcontroller. The system runs on an ESP32 microcontroller with FreeRTOS, the same real-time operating system used in aerospace, automotive, and medical device applications where timing failures could be catastrophic.

The application demonstrates:

Multi-Task Scheduling

Multiple FreeRTOS tasks run concurrently with different priorities and periods. Critical tasks preempt lower-priority work to meet their deadlines.

Hard Real-Time Response

Hardware interrupts provide the fastest possible response—bypassing the scheduler entirely. Used for emergency signals and safety-critical actions.

Timing Verification

A heartbeat LED or serial timestamp log proves the system meets its hard deadlines. Use Wokwi's logic analyzer to verify deterministic behavior with microsecond precision.

Synchronization Mechanisms

Multiple tasks safely share resources using mutexes, binary semaphores, queues, and critical sections—preventing race conditions and data corruption.

Why This Matters: In mission-critical systems, missing a deadline isn't just slow—it's a system failure. A car's brake-control system must respond in milliseconds. A pacemaker must deliver a pulse at the exact right moment. This project proves you understand how to design and verify such systems.

// 03.5 — Project Requirements Met

Application 6 Checklist

This project fulfills all Application 6 requirements for a proof-of-concept real-time system:

✓ Hardware Setup

ESP32 microcontroller with ≥2 external LEDs, ≥1 analog sensor (potentiometer/ADC), and ≥1 momentary input (push buttons). All wired and labeled in Wokwi.

✓ FreeRTOS Tasks & ISRs

≥4 tasks plus ≥1 ISR. At least one task takes variable execution time, simulating realistic workloads. All mapped to company use-case scenarios.

✓ Timing & Deadlines

Each task has a defined period/deadline. Code comments mark tasks as HARD (failure if deadline missed) or SOFT (preferred but not critical).

✓ Synchronization

≥2 distinct mechanisms used (mutex, binary semaphore, queue, critical section). Each is necessary to prevent race conditions and ensure data integrity.

✓ Inter-Task Communication

Internal queue/event group passes data between tasks. External link via UART/serial outputs timestamps and task status to the Wokwi serial monitor.

✓ Determinism Proof

Heartbeat LED blinks at exact intervals, or serial timestamps log every task. Attach Wokwi logic analyzer to the heartbeat GPIO for full-credit verification.

// 05 — Technology Stack

Tools & Platforms

ESP32 FreeRTOS C/C++ Wokwi UART GPIO ADC ISR

ESP32 DevKit V1

Dual-core 32-bit RISC-V/Xtensa microcontroller with built-in WiFi and Bluetooth. Running on Wokwi's online simulator—no hardware required.

FreeRTOS Kernel

Industry-standard real-time operating system. Provides preemptive multitasking, semaphores, queues, and deterministic scheduling—used in aerospace, automotive, and medical.

C/C++ Firmware

Low-level microcontroller programming. Direct register control, interrupt handling, and precise timing loops for hard real-time requirements.

Wokwi Simulator

Cloud-based hardware simulator. Upload code, wire components visually, and test without physical boards. Built-in logic analyzer for timing verification.
