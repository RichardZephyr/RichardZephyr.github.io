Predictive EKG Vitals Monitor

A hard real-time embedded system for cardiac telemetry, ensuring deterministic vitals processing and sub-millisecond emergency response using FreeRTOS and ESP32.

UCF Engineering · Spring 2026 Architecture · FreeRTOS Dual-Core Priority · Preemptive Scheduling
// 01 — System Architecture

Deterministic Patient Monitoring

In clinical environments, a "laggy" heart rate monitor isn't just an inconvenience—it's a system failure. This project implements a robust EKG telemetry monitor on the ESP32 platform, utilizing the FreeRTOS kernel to guarantee that high-priority clinical tasks preempt background operations.

Synchronized Sampling

Vitals are captured via a 50Hz periodic task using Mutexes to ensure data integrity when sharing EKG buffers between the ADC task and the web dashboard.

Clinical Alert Logic

Hard real-time tasks monitor BPM thresholds. If values exceed 100 (Tachycardia) or fall below 60 (Bradycardia), the system triggers an immediate visual and logic-level alarm.

Emergency ISRs

A "Nurse Call" button is mapped to a hardware interrupt (GPIO 23), providing a near-instantaneous response that bypasses the OS scheduler during medical emergencies.

Inter-Task Queues

Standard clinical data is passed to the Web Server task via FreeRTOS Queues, decoupling the time-critical sampling logic from the slower HTTP networking stack.

The Real-Time Advantage: By utilizing vTaskDelayUntil() and preemptive priorities, this system ensures that even if the web server is busy serving a complex dashboard, the EKG sampling task never misses a single heartbeat.
// 02 — Real-World Application

Scenario: Aerospace Health Monitoring

Imagine a long-duration spaceflight mission for NASA. The crew requires autonomous health monitoring systems that can operate reliably without a ground-control link. My project simulates the Life Support Control System (LSCS) for such a mission.

Medical Telemetry Implementation: My system acts as the first line of defense in a deep-space habitat. The Heartbeat Task verifies the LSCS is alive. The Nurse Call ISR simulates an astronaut's emergency tether trigger. The Sensor Tasks simulate oxygen and EKG vitals that must be processed with absolute determinism to prevent hypoxic events.

Task & GPIO Architecture

GPIO Mapping

Component GPIO Pin Type Clinical Role
Alert LED (Red) GPIO 2 Output Active HIGH: Tachycardia/Bradycardia alarm indicator.
Status LED (Green) GPIO 4 Output System Healthy / Within physiological range.
Heartbeat LED GPIO 26 Output 5Hz toggle proving OS scheduler health.
Nurse Call Button GPIO 23 Input (ISR) Hardware interrupt for immediate emergency signaling.
EKG Signal (Pot) GPIO 34 Input (ADC) Simulated analog heart signal (0–4095 range).

FreeRTOS Task Schedule

Task Name Priority Period Type Description
Emergency_ISR HW Aperiodic HARD GPIO 23. Bypasses scheduler to alert clinical staff.
Sampling_Task 4 20ms HARD ADC sampling. Failure results in clinical data loss.
Vitals_Logic 3 100ms HARD Threshold detection for Tachycardia/Bradycardia.
Web_Server 1 Async SOFT Dashboard delivery. Non-critical timing.
